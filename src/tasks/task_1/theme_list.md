# Темы

Допускается подобрать тему по согласованию с преподавателем (например, это может быть тема диплома)

## 1) «Городские обращения 24/7» (GovTech / Service Desk для граждан)

1. **Идея/цель**: прием и обработка обращений жителей (ямы, освещение, мусор), маршрутизация в подрядные организации, контроль сроков.
2. **Стейкхолдеры**: гражданин, оператор, подрядчик, администрация города, аналитик.
3. **Интеграции**: карты/геокодирование, SMS/e-mail/пуш, портал идентификации/учётка, фотохранилище.
4. **NFR-вызовы**: пиковые нагрузки, антифрод/спам, прозрачность статусов, аудит действий.

---

## 2) «Сервис электронных очередей и записи» (для МФЦ/поликлиники/университета)

1. **Идея/цель**: запись на приём/услугу, управление слотами, уведомления и отмены, витрина занятости.
2. **Стейкхолдеры**: заявитель, регистратор, руководитель подразделения, администратор расписаний.
3. **Интеграции**: календарь/расписания, SSO, уведомления, терминалы в холле (опционально).
4. **NFR-вызовы**: консистентность слотов (гонки), доступность, UX, защита персональных данных.

---

## 3) «Платёжный шлюз и биллинг подписок» (FinTech)

1. **Идея/цель**: подписки, автосписания, возвраты, статусы платежей, сверка.
2. **Стейкхолдеры**: клиент, бухгалтерия, служба поддержки, риск-офицер, разработчики.
3. **Интеграции**: эквайринг/банки, антифрод, CRM, учётная система.
4. **NFR-вызовы**: безопасность, идемпотентность, отказоустойчивость, строгий аудит и трассируемость операций.

---

## 4) «Маркетплейс вакансий и стажировок для университета» (EdTech/HRTech)

1. **Идея/цель**: публикация вакансий, отклики, матчинговые рекомендации, статусы, отчёты по трудоустройству.
2. **Стейкхолдеры**: студент, работодатель, карьерный центр, модератор.
3. **Интеграции**: SSO/ЛК студента, e-mail/мессенджер, календарь интервью, аналитика.
4. **NFR-вызовы**: качество рекомендаций (метрики), приватность данных, модерация/спам.

---

## 5) «Система мониторинга и алертинга для продуктовой платформы» (DevOps/SRE)

1. **Идея/цель**: сбор метрик/логов/трейсов, правила алертов, дашборды, инцидент-менеджмент.
2. **Стейкхолдеры**: SRE, дежурный инженер, тимлид, владелец сервиса.
3. **Интеграции**: источники метрик/логов, мессенджеры, on-call расписание, тикет-система.
4. **NFR-вызовы**: высокая нагрузка на ingestion, надежность доставки алертов, минимизация ложных срабатываний.

---

## 6) «Платформа документооборота и согласований» (Enterprise)

1. **Идея/цель**: маршруты согласования (заявки/договора/служебки), версии документов, подписи, контроль сроков.
2. **Стейкхолдеры**: инициатор, согласующий, юрист, бухгалтерия, администратор.
3. **Интеграции**: SSO, хранилище документов, уведомления, справочники оргструктуры.
4. **NFR-вызовы**: юридическая значимость (аудит/неотказуемость), управление доступом, версии/конфликты.

---

## 7) «Каталог данных и управление доступами (Data Governance)» (Data/Platform)

1. **Идея/цель**: каталог датасетов, владельцы данных, политика доступа, заявки на доступ, метаданные и качество данных.
2. **Стейкхолдеры**: дата-аналитик, дата-инженер, data owner, ИБ, аудитор.
3. **Интеграции**: DWH/озёра данных, IAM/SSO, тикетинг, сканеры метаданных.
4. **NFR-вызовы**: безопасность, точность метаданных, прослеживаемость, масштабируемость.

---

## 8) «Система управления складом для e-commerce (WMS-lite)» (LogTech)

1. **Идея/цель**: приёмка, размещение, сборка заказов, инвентаризация, контроль ошибок.
2. **Стейкхолдеры**: кладовщик, супервайзер, менеджер логистики, служба поддержки.
3. **Интеграции**: OMS/ERP, сканеры (как внешнее устройство/ПО), курьерские службы, печать этикеток.
4. **NFR-вызовы**: offline/плохая сеть, скорость операций, точность учёта, удобство для оператора.

---

## 9) «Платформа дистанционного обучения с прокторингом» (EdTech)

1. **Идея/цель**: курсы, тестирование, экзамены с прокторингом, отчёты по успеваемости.
2. **Стейкхолдеры**: студент, преподаватель, проктор, администратор.
3. **Интеграции**: видео/стриминг, SSO, LMS/деканат, античит (опционально).
4. **NFR-вызовы**: приватность, доступность видео, нагрузка, справедливость/ошибки распознавания (метрики качества).

---

## 10) «Сервис управления разрешениями и доступом к API (API Access Management)» (Security/Platform)

1. **Идея/цель**: выдача ключей/токенов, политики доступа, ротация секретов, журналирование, self-service портал для команд.
2. **Стейкхолдеры**: разработчик, DevOps, ИБ, администратор платформы, аудитор.
3. **Интеграции**: IAM/SSO, API gateway, секрет-хранилище, SIEM/логирование.
4. **NFR-вызовы**: безопасность, неотказуемость, надежность, удобство self-service без снижения контроля.
